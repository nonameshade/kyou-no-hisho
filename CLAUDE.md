# CLAUDE.md — 「今日の秘書」開発ルール

**作業前に必ず `HANDOFF.md` を全部読むこと。** プロダクトの全体像・データモデル・処理の地図・これまでの経緯が書いてある。

## 絶対ルール
- **ビルドなし・フレームワークなしの vanilla JS** を維持する。npm依存・トランスパイル・モジュール分割はしない（GitHub Pagesで無ビルド公開のため）。
- ロジックは `app.js`、スタイルは `styles.css`、DOMは `index.html` に集約。ファイルを増やさない。
- コード変更後は必ず `node --check app.js` で構文チェックする。
- **コードを変更したら `sw.js` の `const CACHE = "hisho-vN"` の N を必ず +1 する。** 忘れると利用者に更新が届かない。
- `state` の構造を変えるときは `migrate()` に新バージョンの移行処理を追加し、既存データ（localStorage/スプレッドシート）を壊さない。
- `Code.gs`（Apps Script）を変更したら、手順書に「デプロイを管理→新バージョン」で更新するよう明記する（「新しいデプロイ」はURLが変わるので禁止）。

## 作業の締め方
変更が終わったら、利用者（非エンジニア）向けに次を日本語で簡潔に伝える:
1. GitHubに上書きアップロードするファイル名
2. Apps Scriptの更新要否（Code.gsを触ったときのみ）
3. 変更点の要約

## スタイル/UIの慣習
- カラーは `styles.css` の `:root` のCSS変数を使う。アクセント色 `--accent: #0E7C66`。
- ボタン配置は「キャンセル左下・保存右下」で統一。
- 3つのpointerイベント系統（並べ替え/スワイプ/マーク移動）が併存するので、新しいドラッグ系を足すときは既存と競合しないか確認する。
- 日本語UI。「今日」表記（「きょう」ではなく）。
